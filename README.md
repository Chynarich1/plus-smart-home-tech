# 🏠 Smart Home Technologies — IoT Ecosystem & E-commerce Platform

Комплексная платформа для экосистемы умного дома, объединяющая сбор телеметрии с датчиков и полноценный интернет-магазин устройств.

## 🏗 Архитектура системы

Проект разделен на три независимых функциональных контура, объединенных через Spring Cloud:

### 1. Commerce (Микросервисный интернет-магазин)
Реализован паттерн **Database per Service**. Микросервисы взаимодействуют через **REST (Feign Clients)**.
* `shopping-store` — каталог товаров с управлением категориями и состояниями.
* `shopping-cart` — корзина пользователя с проверкой доступности товаров.
* `warehouse` — управление складом, бронирование товаров и логистика.
* `order` / `payment` / `delivery` — цепочка оформления заказа, процессинг оплаты и расчет стоимости доставки.
* **Circuit Breaker:** Интегрирована устойчивость к сбоям при межсервисных вызовах.

### 2. Telemetry (IoT контур)
Высоконагруженная система сбора и анализа данных с датчиков (освещение, температура, климат).
* `collector` — gRPC-сервер, принимающий события в формате **Protobuf**, кодирующий их в **Avro** и отправляющий в Kafka.
* `aggregator` — формирование снапшотов состояния хаба на основе потока событий.
* `analyzer` — движок сценариев. Сверяет состояние датчиков с пользовательскими правилами и отправляет команды на устройства через gRPC.

### 3. Infra (Общая инфраструктура)
* `config-server` — централизованное управление конфигурациями всех сервисов.
* `gateway` — единый входной шлюз с динамической маршрутизацией и балансировкой нагрузки (**Load Balancer**).
* `discovery-server` (Eureka) — регистрация и обнаружение сервисов.

## 🛠 Технологический стек
* **Связь:** gRPC (Protobuf), REST (Feign), Apache Kafka (Avro).
* **Базы данных:** PostgreSQL (каждый сервис со своей БД), Redis (для агрегации снапшотов).
* **Инструменты:** Docker, Docker Compose, Spring Boot Actuator, JUnit 5, Mockito.

## 🚀 Ключевые инженерные решения
* **Сериализация:** Использование Avro позволило минимизировать объем передаваемых данных в Kafka по сравнению с JSON.
* **Распределенные транзакции:** Логика оформления заказа спроектирована с учетом возможной недоступности сервисов (использование статусной модели заказа).
* **Fault Tolerance:** Настройка Circuit Breaker предотвращает каскадные сбои всей системы при падении сервиса склада или оплаты.
